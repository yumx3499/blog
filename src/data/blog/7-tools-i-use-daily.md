每年都有人写「年度最佳工具清单」，我看了几年，发现一个规律：**大部分清单是用来收集的，不是用来用的。**

所以我今天换一种写法——只说我真正在用的，每天碰的那种。如果你和我的场景类似，直接拿来用就行。如果不像，至少帮你排除几个选项。

---

## 1. VS Code — 写代码的主阵地

**一句话评价**：不是最好的编辑器，但生态无敌。

用了三年了，中间试过 Neovim（太折腾）、Cursor（AI 功能不错但不稳定）、Zed（快但插件少），最后还是回到 VS Code。

核心配置：
- **主题**：One Dark Pro，眼睛舒服
- **字体**：JetBrains Mono，连字很好看
- **插件**：GitLens、Prettier、ESLint、Tailwind IntelliSense

优点：插件生态是真的强，几乎什么语言都能写。
缺点：吃内存，开多了标签页会卡。但比 JetBrains 系列还是轻量不少。

适用场景：所有写代码的场景。如果你还在用纯文本编辑器，是时候升级了。

## 2. Raycast — 替代 Spotlight 的效率神器

**一句话评价**：macOS 上的 Alfred，但免费版就够用。

这个工具的核心价值是：**让你的手不离开键盘**。

我用它做的事：
- 打开应用（比点击 Dock 快 10 倍）
- 计算（不用打开计算器）
- 搜索文件（比 Finder 快）
- 剪贴板历史（这个功能就值回票价）
- 快速查文档（有插件支持各种 API 文档）

优点：免费版功能就很全，插件生态不错。
缺点：macOS only，Windows 用户可以用 PowerToys 替代。

适用场景：任何不想把手从键盘移到鼠标的时刻。

## 3. Obsidian — 第二大脑

**一句话评价**：Markdown 笔记的天花板。

我用 Obsidian 管理一切：
- 会议笔记
- 项目文档
- 读书笔记
- 灵感碎片
- 甚至这个博客的草稿也是在 Obsidian 里写的

核心优势：
- **本地 Markdown 文件**：数据在你手里，不用担心服务关闭
- **双向链接**：笔记之间可以互相链接，形成知识网络
- **插件生态**：社区插件 1000+，几乎什么功能都能找到

缺点：初始设置有点复杂，需要花时间调教。但一旦调好，效率提升是指数级的。

适用场景：需要长期积累知识的人。如果你只是偶尔记个 TODO，用系统备忘录就够了。

## 4. Terminal + Oh My Zsh — 命令行美化

**一句话评价**：程序员的面子工程，但真的有用。

我知道很多人觉得终端能用就行，不用折腾。但一个好的终端配置真的能提升效率：

- **Oh My Zsh**：自动补全、命令别名、主题
- **主题**：我用的是 agnoster，显示 git 分支很方便
- **别名**：`gs` = `git status`，`gp` = `git push`，省了多少次敲键盘

```bash
# .zshrc 里的常用别名
alias gs="git status"
alias gc="git commit"
alias gp="git push"
alias ll="ls -la"
alias dev="cd ~/projects && npm run dev"
```

优点：省时间，看着也舒服。
缺点：配置起来有点繁琐，但网上教程很多。

适用场景：每天都用终端的人。如果你一个月才开一次终端，那算了。

## 5. Notion — 项目管理和协作

**一句话评价**：虽然被说"大而全"，但确实什么都能干。

坦白说，Notion 不是最好的任何东西——不是最好的笔记软件，不是最好的项目管理工具，不是最好的数据库。但它是什么都能干的那一个。

我用 Notion 管理：
- 项目看板（Kanban 视图）
- 团队文档
- 内容日历（规划博客发布）
- 读书/观影清单

优点：灵活，协作体验好。
缺点：离线模式约等于没有，加载速度一般。

适用场景：需要和别人协作的场景。如果是纯个人使用，Obsidian 更好。

## 6. Arc Browser — 浏览器的重新想象

**一句话评价**：如果你受够了 Chrome 的标签页地狱。

Arc 把浏览器从"打开网页的工具"变成了"工作空间"：

- **垂直标签栏**：终于不用在 50 个标签里找目标了
- **Easels**：可以把网页内容钉在画板上，做对比研究特别好用
- **Spaces**：按场景分组标签页，工作/娱乐/学习分开

优点：重新定义了浏览器的交互方式，效率提升明显。
缺点：基于 Chromium 但有些网站兼容性一般，而且目前只有 macOS。

适用场景：每天打开浏览器超过 50 次的人。如果你只是偶尔查个资料，Chrome 也挺好。

## 7. Terminal JSON viewer — `jq` + `bat`

**一句话评价**：让命令行输出变得能看。

两个小工具，但每天都在用：

**`jq`** — JSON 格式化
```bash
# 之前：curl 返回一坨乱麻
curl https://api.example.com/data

# 之后：结构清晰
curl https://api.example.com/data | jq '.'
```

**`bat`** — 替代 `cat`，有语法高亮
```bash
# 之前
cat config.ts

# 之后：自动高亮、显示行号
bat config.ts
```

优点：小而美，装了就回不去。
缺点：需要习惯命令行，非开发者可能用不到。

适用场景：每天和 API/JSON 打交道的人。
