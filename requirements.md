# 📌 个人博客网站 — 需求分析文档

> 扒皮出品 | 2026-03-18
> 
> 一句话：别做"大而全"，做一个能持续产出的博客，比什么都重要。

---

## 1. 博客定位分析

| 类型 | 核心价值 | 典型用户 | 更新频率 | 技术深度 |
|------|----------|----------|----------|----------|
| **技术博客** | 展示技术能力，吸引工作/合作机会 | 程序员、技术创业者 | 中（月更） | 高 |
| **作品集** | 展示设计/开发作品，接单用 | 自由职业者、设计师 | 低（项目制） | 低 |
| **随笔博客** | 记录生活、观点输出，个人表达 | 任何人 | 高（周更） | 中 |
| **知识管理** | 个人 Wiki，笔记沉淀 | 学习者、研究者 | 持续 | 高 |

### 🎯 我的建议

**先做「技术博客 + 轻量作品集」混合型。**

理由：
- 老板搞自动化咨询，技术博客能直接转化客户
- 作品集部分放项目案例，增加可信度
- 随笔可以有，但别当主力——产出太随机

## 3. 技术路线建议

### 方案对比

| 方案 | 类型 | 学习成本 | 性能 | 部署 | 扩展性 | 推荐指数 |
|------|------|----------|------|------|--------|----------|
| **Astro** | SSG | 低 | ⭐⭐⭐⭐⭐ | Vercel/Netlify 一键 | 中 | ⭐⭐⭐⭐⭐ |
| **Next.js SSG** | SSG | 中 | ⭐⭐⭐⭐⭐ | Vercel 一键 | 高 | ⭐⭐⭐⭐ |
| **Hugo** | SSG | 低 | ⭐⭐⭐⭐⭐ | 任意静态托管 | 低 | ⭐⭐⭐⭐ |
| **Hexo** | SSG | 低 | ⭐⭐⭐⭐ | GitHub Pages | 低 | ⭐⭐⭐ |
| **WordPress** | 动态 | 低 | ⭐⭐ | 需要服务器 | 高 | ⭐⭐ |
| **Strapi** | Headless CMS | 高 | ⭐⭐⭐ | 需要服务器 | 高 | ⭐⭐ |

### 🏆 推荐方案：Astro

**为什么选 Astro：**

1. **天生为内容而生** — 博客是 Astro 的主场，不是副业
2. **零 JS 默认** — 打开速度碾压 React/Vue 框架
3. **Markdown 就是 CMS** — 写文章就是写 .md 文件，不用后台
4. **组件复用** — 未来加功能（比如作品集页面）用组件就行
5. **部署简单** — `vercel deploy` 一条命令

**为什么不选其他：**

- **WordPress** — 太重了，维护成本高，需要数据库，个人博客杀鸡用牛刀
- **Hexo/Hugo** — 好用但扩展性差，以后想加功能（比如动态效果）很痛苦
- **Next.js** — 万能但博客场景过度设计，学习曲线陡

## 4. 用户画像假设

| 画像 | 场景 | 会看什么 | 痛点 |
|------|------|----------|------|
| **技术同行** | 搜到某篇技术文章点进来 | 代码示例、踩坑记录 | 要能快速定位到相关代码 |
| **潜在客户** | 被推荐来看看你靠不靠谱 | 关于页、项目案例、专业文章 | 要一眼看出你专长什么 |
| **自己** | 记录/查阅之前写过的东西 | 搜索、分类 | 要能快速找到历史文章 |
| **搜索引擎路人** | Google/百度搜到 | 第一段就要有料 | 文章标题和摘要要SEO友好 |

## 6. 最终推荐方案

```
技术栈：Astro + TypeScript
模板：astro-paper（改造）
部署：Vercel（免费）
内容管理：Markdown 文件（本地写好 push 上去）
域名：先用 vercel.app 子域名，后续可绑自定义域名
```

### 第一版目标（MVP）

1. ✅ 首页 — 文章列表 + 关于我侧边栏
2. ✅ 文章页 — Markdown 渲染 + 代码高亮 + TOC
3. ✅ 关于页 — 个人介绍 + 联系方式
4. ✅ 分类/标签筛选
5. ✅ 暗色模式
6. ✅ 基础 SEO

**预计工期：3-5 天**（码神 + 调调配合）
